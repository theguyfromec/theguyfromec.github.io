Sales / CS
Inbound / outbound lead
Leads from website, referrals, or Apollo IO + Claude outbound agent (Graham). SDR should create HubSpot record here.
Request trial access for prospect
Sales/CS requests access to trial.cloud.snapstream.com. Manual process, no automation.
Sales initiates quote request
Sales decides to quote. Triggers PandaDoc quote creation. Owner: Kyle T.
Create deal, attach docs, add line items
All manual. Deal created in HubSpot, PandaDoc copy attached, line items entered by hand. Order of operations not enforced.
System install, training, go-live
Equipment shipping, DirecTV setup, SnapStream install + config, product training, admin training. 0-30 day timeline.
CSM reviews renewal, engages customer
120-day workflow auto-assigns CSM task (due 3 days). CSM reviews, sends renewal comms. 60-day second checkpoint. Auto-renew at day 59 for standard renewals.
Mark customer churned
Sales/CS marks customer as churned in HubSpot. Triggers email and Freshdesk ticket to Daryl.
PandaDoc
Create quote, send for e-signature
Sales creates quote manually in PandaDoc. Sent to customer for e-signature. No templates documented. $5K/year cost.
Executed copy used as deal attachment
Signed PandaDoc is manually attached to HubSpot deal record. No automated handoff.
Renewal quote (if needed)
Some renewals require a new PandaDoc quote. Template reportedly missing (Jill, Jan 2026). Some renewals require customer PO instead.
HubSpot
Contact record created
Should be entry point. Sometimes created retroactively after PandaDoc quote. ~60K records, ~200 active, never cleaned.
Track trial status
Should track trial stage in HubSpot. No auto-provisioning (API deprecated). Commerce Hub blocked by Abby/Stripe.
Closed Won workflow fires
Workflow ID 4580384. Sends deal info to Freshdesk, posts to Slack "Sales Deal Updates". Syncs Product Type, ARR, Service End Date to company record.
Line items sync to Maxio
Limited connector (previous team, poorly set up, no current owner). Exact sync fields undocumented. $166K unprocessed orders in Maxio.
HubSpot Projects: 13-task onboarding template
Replacing GuideCX (Jill, March 2026). 0-30 day timeline. Tasks: kick-off, cloud setup, DirecTV, training, CSM transition. Purchase type "New" auto-creates; Expansion/SysReplace not wired yet.
120-day renewal workflow auto-creates deal
Triggers when Next Renewal Date < 120 days. Auto-creates deal in "Existing Business" pipeline. Posts to Slack, emails deal owner. CRITICAL: Next Renewal Date is manual (SPOF).
Churn email trigger
Customer marked churned triggers email to Freshdesk. No churn reason tracking. No vol/invol differentiation in workflow.
Maxio
Receive line items, invoice customer
Receives synced line items from HubSpot. Accounting runs Invoice Green Light checklist via Freshdesk ticket. Hard constraint: line items + contract dates must be accurate.
Involuntary churn: 60+ days past due
Collections escalation: 30d first notice, 45d second, 50d escalation, 60d suspension warning. High-value accounts (>$100K) get different treatment. Leads to suspension/termination.
QuickBooks Online
Daily sync from Maxio
Invoices and payments sync daily. Known issues: multicurrency mismatch (QBO enabled, Maxio not), Piping Technology status conflict. Closing date 12/31/2025.
Freshdesk
Onboarding ticket auto-assigned to Paco
HubSpot Closed Won email creates Freshdesk ticket. Auto-assigned to Paco. Also triggers on renewal close.
Invoice Green Light ticket
Accounting receives this ticket to trigger billing checklist. Created from HubSpot automation on deal close.
Onboarding ticket (Paco)
Same trigger as Stage 4. Paco manages onboarding execution via Freshdesk ticket alongside HubSpot Projects.
Churn ticket auto-assigned to Daryl
HubSpot churn email creates Freshdesk ticket for Daryl. Post-ticket steps undocumented. Kyle floated moving termination to Paco.
Trial Server
trial.cloud.snapstream.com
Single shared instance. Users added manually. Purgatory group for inactive users (preserves stats). No tenant isolation. Future: paid trial via Commerce Hub.
